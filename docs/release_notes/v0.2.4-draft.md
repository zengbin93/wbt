# wbt v0.2.4 (draft)

> Release date: TBD
> Crate tag: `crate-v0.2.4` · Python tag: `v0.2.4`

## Summary

MINOR release adding a single new method to `WeightBacktest`:

- `wb.is_good_strategy(mode, target_vol, max_dd_threshold, min_year_days, recent_days) -> dict`
  — objective verdict on whether a strategy is worth pursuing, with two complementary modes
  (`"history"` / `"recent"`) and an explicit non-overlapping window guarantee in `recent` mode.

No removals; no breaking changes to existing APIs.

## Additions

- **`WeightBacktest.is_good_strategy(...)`** (Rust core in `src/core/is_good_strategy.rs`; exposed via PyO3 in `src/lib.rs::PyWeightBacktest`; Python facade in `python/wbt/backtest.py`):

  - `mode="history"` — pass when every **complete** calendar year (>= `min_year_days` trading days)
    satisfies `abs_return > 0` OR `volatility-normalized long-alpha > 0`, AND the full-sample
    long-alpha max drawdown is below `max_dd_threshold`.
  - `mode="recent"` — pass when the last `recent_days` window satisfies the same return condition
    AND its long-alpha max drawdown is strictly less than both `max_dd_threshold` and the
    **history-excluding-recent** max drawdown. The two windows are guaranteed to be disjoint;
    if there is no sample left after excluding the recent window, the method reports
    `history_window_empty=true` and `is_good=false` with `reason="history window empty (sample too short)"`.

  Adjustable parameters with defaults: `target_vol=0.20`, `max_dd_threshold=0.20`,
  `min_year_days=120`, `recent_days=252`. Top-level dict keys are English `snake_case`
  (e.g. `is_good`, `mode`, `yearly_metrics`, `recent_alpha_max_drawdown`,
  `history_alpha_max_drawdown_excl_recent`), independent from the existing Chinese-keyed
  `*_stats` family. The method does **not** participate in `STATS_FIELD_ORDER`.

- **`hashmap_to_pydict` helper extension** (`src/lib.rs`): now recursively converts
  `serde_json::Value::Bool` / `Array` / `Object` (in addition to the existing `Number` / `String`),
  so nested structures like `yearly_metrics` (list of objects) cross the FFI boundary correctly.
